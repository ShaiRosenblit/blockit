## Candidate L7-1 — Clear-Triggered Column Rotor
- Lens: 7 (Topological twist)
- Generated by: GPT-5.3 (codex)
- Rule changes: Puzzle baseline (finite tray + target match), plus: after any placement that clears at least one line, rotate all board columns right by 1 (`newCol = (oldCol + 1) mod 8`) before the next move; no rotation on non-clear placements.
- Win condition: `tray.length === 0 && boardMatchesTarget(board, target)`.
- Lose condition: `(no legal placements for all tray pieces) || (tray.length === 0 && !boardMatchesTarget(board, target))`.
- New question per placement: "Do I trigger a clear now to rotate every occupied cell/right-edge gap by +1, or avoid a clear to preserve current column alignment for later pieces?"
- Dual-purpose mechanic: line clear (target progress + topology clock tick).
- Antagonist: prefill cells that drift into future target lanes when rotation is mistimed.
- Negative-space constraint: specific destination columns must remain empty after each possible post-clear rotation phase.
- Order coupling: the same placements in different clear timings produce different column phases and therefore different legal futures.
- Topology rule: rotation amount is deterministic and stateful: `phase = (phase + 1) mod 8` exactly when `clearedRows + clearedCols > 0`; occupancy map is transformed by that phase shift in O(64).

- Generated by: GPT-5.3 (codex)
- Rule changes: Puzzle baseline, plus a `clearAxis` state starting at `rows`; when `clearAxis=rows`, only full rows clear; when `clearAxis=cols`, only full columns clear; after any clear event, toggle axis (`rows <-> cols`).
- Win condition: `tray.length === 0 && boardMatchesTarget(board, target)`.
- Lose condition: `(no legal placements for all tray pieces) || (tray.length === 0 && !boardMatchesTarget(board, target))`.
- New question per placement: "Should this move clear under the current axis and flip the next clear geometry, or should I avoid clearing so the same axis remains for another setup move?"

- Antagonist: axis mismatch between current board density and target-required empties.
- Negative-space constraint: cells that would accidentally complete the inactive axis must stay empty until the axis flips.
- Order coupling: identical piece set with different clear timing yields different axis sequence, changing which lines can ever be deleted at each step.
- Topology rule: `clearAxis` is a 1-bit deterministic automaton driven only by whether the last placement cleared; reducer checks one orientation of line clear per move in O(64).

- Rule changes: Puzzle baseline, plus one fused row pair each turn: `(0,1)`, `(2,3)`, `(4,5)`, `(6,7)` cycling every placement; for line-clear detection only, a fused pair counts as one logical row that clears when both physical rows are fully filled together; non-fused rows clear normally if complete.

- Lose condition: `(no legal placements for all tray pieces) || (tray.length === 0 && !boardMatchesTarget(board, target))`.

- Negative-space constraint: in a soon-to-be-fused pair, one row may need intentional holes so the pair does not clear prematurely.
- Order coupling: same placements with a one-turn offset encounter different fusion phases and therefore different clear outcomes.
- Topology rule: fusion index is deterministic `phase = moveIndex mod 4`; clear detector reads phase and evaluates one paired-row predicate plus remaining standard predicates in O(64).

- Rule changes: Puzzle baseline, plus a vertical seam between columns 3 and 4; every third placement opens seam mode for exactly one move (`moveIndex % 3 === 0`): during open mode, any piece cell that would cross board edge horizontally wraps (`col = (col + 8) mod 8`) if the local piece segment crosses the seam; when closed, normal bounds apply.
- Win condition: `tray.length === 0 && boardMatchesTarget(board, target)`.
- Lose condition: `(no legal placements for all tray pieces) || (tray.length === 0 && !boardMatchesTarget(board, target))`.

- Antagonist: seam-open turns, which can create hard-to-undo wrapped occupancy in far columns.
- Negative-space constraint: seam-adjacent cells must often stay empty before an open turn so wrapped cells have legal landing spots.
- Order coupling: using or skipping seam-open turns changes which pieces can exploit wrap geometry later.
- Topology rule: seam state is deterministic by move index; coordinate projection is a pure function in placement validation/application and stays O(pieceCells + 64).

## Candidate L7-8 — Deterministic Hinge Fold
- Lens: 7 (Topological twist)
- Generated by: GPT-5.3 (codex)
- Rule changes: Puzzle baseline, plus a hinge line that alternates each move: odd turns fold across vertical center (columns mirrored), even turns fold across horizontal center (rows mirrored); after line clears, if one side of current hinge is empty in a mirrored pair and the other is occupied, occupied cell copies into the empty partner (never overwrites occupied cells).
- Win condition: `tray.length === 0 && boardMatchesTarget(board, target)`.
- Lose condition: `(no legal placements for all tray pieces) || (tray.length === 0 && !boardMatchesTarget(board, target))`.
- New question per placement: "Should I clear now so hinge-copy runs on this turn's fold axis and duplicates into the partner half, or delay so copying happens on the other axis?"
- Dual-purpose mechanic: clear event (removal plus controlled duplication along current fold axis).
- Antagonist: hinge-copy propagation, which can repopulate areas the player just cleaned if timed poorly.
- Negative-space constraint: mirror-partner cells that must remain empty in target cannot be adjacent to one-sided occupancy on the turns when that hinge axis is active.
- Order coupling: clearing on odd vs even turns applies different fold axes, yielding different copied occupancy and future legal moves.
- Topology rule: hinge axis = `moveIndex parity`; copy pass runs deterministically after clear-resolution with a single O(64) scan over mirrored coordinate pairs.
