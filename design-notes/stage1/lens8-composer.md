# Lens 8 — Inversion (Composer-2-fast)

**Summary:** Eight mode sketches invert existing primitives (clears, legal placement, tray consumption, and clear resolution) so each `PLACE_PIECE` and related actions ask a non‑Classic question—avoiding a full line, satisfying a global flood region, growing one polyomino from a seed, or choosing discards, axis-locked clear resolution, blocker conversion, or a meter hit—while keeping the 8×8 board, `PIECE_CATALOG`, 3‑slot tray, and standard `applyPlacementAndClear` pipeline with `mode === '...'` branches and forward‑sim solvability. Blockit candidates are in mechanics‑speak; implementation would still need per‑mode generator filters and disease checks (e.g. discard RNG must be designer‑controlled, not player‑invisible noise).

## Candidate L8-1 — Anti-line: full row/column completion ends the round
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (same target overlay): after `placePiece` + `detectCompletedLines`, if **any** row or column is fully occupied (pre‑clear state), the round is a **lose** before line removal runs; or equivalently, **skip** the clear step entirely and treat a completed line as illegal (generator must only emit instances solvable with **zero** full rows/columns at any intermediate state). Standard rotation/placement; tray refill as base mode.
- Win condition: `boardMatchesTarget` is true, round not lost, and **no** `PLACE_PIECE` has ever produced a state with a full row or column (8 filled cells) before clears—matched to a forward‑sim that never steps through such a state.
- Lose condition: any placement would create a full row/column, or a full row/column exists after a placement; or** Classic-style** `no valid placement` with non‑empty tray (per existing move test).
- New question per placement: **“Where can I add cells without ever completing a full line, while still working toward the target and preserving future holes?”** (differs from Classic’s unconstrained “where do I place?” and from Puzzle that can use clears as a deletion verb.)
- Dual-purpose mechanic: **Line detection** (normally coupled to **rewarding** clear) inverts to **forbidden** full-line state; the same “is this line full?” check both gates progress and defines loss.
- Antagonist: the **8‑in‑a-line** threshold on any row or column of *occupied* (placeable) cells.
- Negative-space constraint: **holes and breakpoints** in rows/columns that must never simultaneously fill; *where* you do not place is as load-bearing as where you do.
- Order coupling: earlier placements set **forced gaps**; later pieces must not close any row/column, so the **sequence** of fills changes which continuations are legal.
- The inversion: **Full line** flips from **“trigger clear (good in Classic) / deletable junk (Puzzle)”** to **“immediate fail state (bad).”**

## Candidate L8-2 — Perimeter-anchored placement
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (target + prefill optional): `canPlacePiece` is restricted so that, in board coordinates, **at least one** cell of the placed footprint lies on **r = 0, r = 7, c = 0, or c = 7**. Rotation/placement and row/column clears after placement unchanged.
- Win condition: `boardMatchesTarget` is true; round not lost; all placements in the solution path respected the anchor rule.
- Lose condition: no legal anchored placement for any tray piece (existing “no moves” end); or violation cannot occur if predicate is enforced in `canPlacePiece`.
- New question per placement: **“Which border segment does this shape attach to, and how does that anchor the interior fill relative to the target?”** (Not reducible to “where on the free board?” because the anchor constraint is a separate predicate.)
- Dual-purpose mechanic: **Board edge** (normally soft boundary) doubles as a **legality bit** for every piece instance.
- Antagonist: the **outer ring** (only eight cells wide—corners are cheap doubly) competing for *which* border attachment blocks the next shape.
- Negative-space constraint: **Interior-only** holes may be **unplaceable** until the perimeter is used to “carry” a shape inward in the right order.
- Order coupling: different border attachments change **which interior regions** remain reachable under the same target.
- The inversion: **Placement interior-first freedom** flips to **at least one cell must touch the boundary** every time.

## Candidate L8-3 — Monolith: 4‑connected placenta from a seed
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle**: fix a **seed coordinate** (from generator) on an empty cell. First legal placement’s footprint must **include the seed**; every later placement’s footprint must **4‑connect** to the **union of all previously placed living cells** (scaffold connectivity). `applyPlacementAndClear` unchanged; if a clear splits connectivity, the post‑clear occupancy graph defines connectivity for the next check (or define connectivity on *placed* color cells only—one consistent rule in reducer, O(n²) BFS per move).
- Win condition: `boardMatchesTarget` and seed rule satisfied for the full play trace.
- Lose condition: `no valid placement` that preserves connectivity+seed; or (if defined) **disconnect** of living cells (if that variant is used—clear must be defined not to end in illegal connectivity).
- New question per placement: **“Does this addition keep one connected mass while clearing junk, and which attachment point preserves routes for the remaining tray?”**
- Dual-purpose mechanic: **Clears** both remove prefill/reshape territory **and** may **change** which cells count for connectivity—same primitive, two pressure directions.
- Antagonist: the **growing clump**—it can **wall off** the board center if attachments are wrong.
- Negative-space constraint: **empty** cells that must stay empty for the target are also **gaps in the 4‑connected graph** the player may need to bridge later.
- Order coupling: **branching order** of attachments changes **hole topology** and corridor width for later families.
- The inversion: **Disjoint placements allowed** (Classic) flips to **all placed material must be one 4‑connected component** (after clears).

## Candidate L8-4 — Flood-from-edge: legal region grows from the border
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle**: maintain a set **R** of cells (bitmask 8×8, O(64)). Initialize **R** = { all cells on r∈{0,7} or c∈{0,7} } minus any **static blockers** from generator. A placement is legal iff **every** cell of the footprint lies in **R** and obeys `canPlacePiece`. After a successful `applyPlacementAndClear`, set **R := R ∪ { cells 4-adjacent to any occupied live cell }** (and subtract blockers). Generator forward‑sim only picks placements with footprint ⊆ R at each step.
- Win condition: `boardMatchesTarget` and every placement in the win path was subset‑legal for its step’s **R**.
- Lose condition: `no` legal move with footprint in current **R**; standard game-over probe uses the restricted `canPlace`.
- New question per placement: **“Given the current allowed shell, do I claim edge-adjacent space now or delay so the next ring stays usable for a larger polyomino?”** Distinct from L8-2: here **entire** shape must be inside a **growing** region, not a single border touch.
- Dual-purpose mechanic: **Occupancy** both scores toward target **and** **expands R**; filling “early” can speed access or waste ring capacity.
- Antagonist: **R’s frontier** and **static blockers** on the edge that pinch **R**’s growth.
- Negative-space constraint: **Cells not yet in R** are **unplaceable** even if empty—stronger than “don’t put here for strategy.”
- Order coupling: **Ring expansion order** changes which **large** pieces can ever fit in **R** before it is consumed by clutter.
- The inversion: **“Anywhere empty is eventually placeable”** flips to **“only the flooded shell is placeable until it grows.”**

## Candidate L8-5 — Burn budget: discard is a first-class move
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (no Classic scoring): add action **DISCARD** (or hold hotkey) on active tray index: remove that `PieceShape` from the slot, increment `discardsUsed`, **refill that slot** from a **deterministic** post‑shuffle queue (same as generator’s order after forward‑sim) or from `generateClassicTray` with a **capped redeal**. **Lose** if `discardsUsed > B` (difficulty). Placement unchanged.
- Win condition: `boardMatchesTarget` and `discardsUsed ≤ B`.
- Lose condition: `discardsUsed > B`; or no **place** and no **discard** that keeps discards in budget (if discard is optional); or standard no-move.
- New question per placement: **“Do I place this shape, or pay a discard to trade it for a different family from the same finite queue?”** This is not Classic’s “where” alone—it adds **resource** choice on the tray.
- Dual-purpose mechanic: **Tray slot** (normally “asset to place”) inverts to **“asset you may pay to delete”**; refill keeps **finite lookahead** pressure.
- Antagonist: the **countdown** `B − discardsUsed` and **bad local triples** in the tray.
- Negative-space constraint: **Unused** discards in late game can mean the **wrong** shape never leaves the queue.
- Order coupling: **discard / place** interleaving changes **which** pieces remain in the final multiset vs Puzzle’s fixed bag after shuffle.
- The inversion: **“Tray pieces are only for placing”** flips to **“tray pieces may be destroyed to manipulate the hand.”**

## Candidate L8-6 — Axis-gated clear
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (or Classic): add `clearAxis: 'row' | 'col'` in `GameState`. On each **`PLACE_PIECE` only**: after `placePiece`, run `detectCompletedLines` but **only remove lines matching `clearAxis`** (only row clears if `'row'`, only column clears if `'col'`); then `clearAxis` toggles. **Rotations** do not advance `clearAxis` (or define explicitly—either is O(1)). Other mode logic unchanged.
- Win condition: `boardMatchesTarget` (if puzzle) or Classic score/level rules if not.
- Lose condition: standard no-move; optional fail if a **stuck** full line of the *inactive* axis can never be cleared (designer can ignore if generator forbids).
- New question per placement: **“This step only rows (or only cols) can clear—where do I set up line completion for the *active* axis on *this* turn, knowing the next turn flips?”**
- Dual-purpose mechanic: **Line completion** inverts: **undesirable** for the *inactive* axis (clogs) but **necessary** for the *active* axis; same fill pattern, different legality of resolution.
- Antagonist: **Complimentary full lines** of the **wrong** axis that sit **unresolved** and consume space.
- Negative-space constraint: **Cells that only clear on the other axis** must be managed across **two** placement beats.
- Order coupling: **Parity of move index** (via `clearAxis` sequence) changes whether a line is a **hazard** or a **tool**.
- The inversion: **“Both row and col clears on any full line set”** flips to **“Only one family of lines may fire per placement, alternating.”**

## Candidate L8-7 — Clear-to-blocker: lines become permanent unplaceable cells
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (or Classic): when `applyPlacementAndClear` would run **clear lines**, instead of **emptying** those cells, mark them **static blockers** (existing `BLOCKER_COLOR` / sentinel) **deterministically** for every cell in the union of cleared lines. Blockers are **not** occupiable. No randomness. Generator simulates this “clear = carve barrier” forward.
- Win condition: `boardMatchesTarget` with blocker cells treated as off-limits; target may **exclude** those cells; targets must be chosen so solvable.
- Lose condition: no legal placement; or **inadvertently** block required target (player error).
- New question per placement: **“Should I *trigger* a line now to post blockers that fence the interior, or avoid triggers to keep placement volume?”** Clears no longer *free* area—they *consume* it.
- Dual-purpose mechanic: **Line clear** inverts: **territory removal** in Classic **vs** **permanent obstacle seeding** here—**same** detection pipeline, **opposite** post‑state.
- Antagonist: **Blocker** cells that **shrink** the free grid **monotonically** (unless combined with a mode that removes blockers—**not** in this sketch).
- Negative-space: **Blocker** cells are **hard** negative; **larger** cleared sets mean **more** of the 8×8 is **unusable** forever.
- Order coupling: **When** a line finally completes changes **where** the **walls** go for later **routing**.
- The inversion: **Cleared line cells become empty (good)** flips to **cleared line cells become blockers (clutter).**

## Candidate L8-8 — Purity meter: clears eat the win budget
- Lens: 8 (Inversion)
- Generated by: Composer-2-fast
- Rule changes: vs **Puzzle** (or Classic with hidden meter): add `purity` (integer, e.g. starts at `P0`). On each **row/col clear** event (after `applyPlacementAndClear` resolves), `purity -= w_r + w_c` for counts of **lines cleared** (weights by difficulty), clamp to ≥0. **Win** (puzzle) requires `boardMatchesTarget` **and** `purity ≥ P_min` (e.g. 1), or a floor by difficulty. Display can reuse score bar with different label (minor copy change, no new component).
- Win condition: `boardMatchesTarget` (if puzzle) **and** `purity ≥ P_min` at end; intermediate drops allowed.
- Lose condition: if `purity` hits 0 **and** a rule says **immediate loss on 0** (optional), or end-of-puzzle check fails; or no moves.
- New question per placement: **“Which necessary clears (e.g. to delete prefill) are worth the meter hit vs leaving debris that blocks a cheaper path?”** Distinct from L8-1: **here some clears are mandatory**; the **meter** is the tension, not binary illegality of full lines.
- Dual-purpose mechanic: **Clears** are still **necessary** for Puzzle prefill (good) but **deplete** `purity` (bad for end condition)—**explicit tradeoff** on the same event.
- Antagonist: the **purity** budget depleting toward **P_min**; optional **pre-fill** volume that *forces* clears.
- Negative-space: **Debris** you leave to **avoid** clears may block target—**negative** of clear-spam and **negative** of junk.
- Order coupling: **Order of clears** changes **cumulative** `purity` before the **last** target cells lock in.
- The inversion: **Line clears = score gain / progress** (Classic) flips to **line clears = spend from a win-linked meter** in modes where **high purity** is required to **win**.
